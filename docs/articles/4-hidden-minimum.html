The Minimum Effective Dose (to continue the study)

Uploaded on 2025 x. xth

I've rewritten this article a couple of times because it's about a personal matter. By this I mean that it's a frustration that I can't do anything about short of accepting that the frustrating thing happens.

95% of my work is scripting studies. I script them based on questionnaires which don't come from a single source - which is where my frustration comes from. You see, I can't provide feedback on the questionnaire until it went through a whole boatload of people and was deemed "Ready for Professional Services".

That is me.
I am Professional Services.
Allow me to service this questionnaire professionally.d

Based on what I've heard from other developers, when you have a product that's used by clients and

Let me present you with a scenario - you have a study and you need some people to complete it. It has two important sections - one about cars and one about smartphones (which aren't mutually exclusive). Your client / boss / whoever gave you specific instructions on who should see those sections. You can see the relevant part of the questionnaire below.

You must select at least one answer for each question shown in order to progress.

  • ...

  • Q3. Do you own any of the following?
    Select all that apply

  • Show if Code 1 (Car) is selected in Q3

    Q4. What brand is your car?
    If you have multiple cars, please select the brand of the car you use the most.

  • Show if Code 6 (Smartphone) is selected in Q3

    Q5. What brand is your smartphone?
    If you have multiple smartphones, please select the brand of the one you use the most.

  • If the respondent:

    • didn't select Code 1 or 6 in Q3, they are screened out
    • saw Q4 and selected Code 4 and didn't see Q5, they are screened out
    • saw Q5 and selected Code 3 and didn't see Q4, they are screened out
    • saw Q4 and selected Code 4 and saw Q5 and selected Code 3, they are screened out
    • saw Q4 and didn't select Code 4 and didn't see Q5, they can continue and complete the Car section
    • saw Q4 and didn't select Code 4 and saw Q5 and selected Code 3, they can continue and complete the Car section
    • etc...

The problem with this is that it's too fucking wordy. A goddamned condition spaghetti because somebody didn't sit down and think.

A very important skill that I don't see discussed anywhere (maybe because I'm not looking, but regardless) is the distillation of information. Taking two pages of legalese and explaining it in two paragraphs instead, taking a look at the description of ten requested features for an app and realising that it's actually three features described in just as many ways (and one for the lolz).

More pertinently, this is very useful for the questionnaire section above, because it can be distilled into the below:

  • If the respondent:
    • saw Q4 and didn't select Code 4 they may continue and complete the Car section
    • saw Q5 and didn't select Code 3 they may continue and complete the Smartphone section
    • otherwise they must be screened out

This is also useful in general programming, though it shouldn't always be implemented like this.

If, for example, you're working on a complex submission form then you have to check what's wrong rather than what's right so that you can show the user relevant feedback.

If, however, you're working on some hidden function that needs to run fully only in a few specific cases, go forth and distill. :)
