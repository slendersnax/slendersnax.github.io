"Why doesn't the 'Dark' button do anything for me?"

Prog. Art. Home

Uploaded on 2023 Sept. 7th

Long story short

I used a media query to check if a user prefers-color-scheme: dark but didn't integrate it into the pure CSS light / dark mode switcher (made using :checked tomfoolery).

Long story long

I really wanted to implement a light / dark mode switcher with nothing but HTML and CSS, and I managed! It's not complicated, and it means the html code has a few more <div> elements than I initially wanted to have, but it works great.

Just FYI, I didn't come up with this myself (also if you want you can check out the page structure with the DevTools and see if you can figure it out by yourself first).

The Method and The Issue

I have almost all the contents of the page wrapped in two divs, which are needed for mobile responsiveness and dark mode. There's a <checkbox> on the same level as the first div, which means that with the :checked and ~ selectors we can change the whole page's style based on whether the checkbox is checked or not.

However, I also didn't want to blind anyone who prefers-color-scheme: dark, so I made a media query for that. But because the condition for this query is always true, it won't take into account the checkbox and stay in dark mode.

There are workarounds for this, and I'm planning on implementing one of them - this sounds cool, but I unless I figure out a way to do it more elegantly I will just copy the light / dark switcher CSS into the media query.

I could also (and might):

  • use JavaScript to click the checkbox if the user prefers dark mode
  • hack around until I find a way to activate the checkbox with pure CSS as well based on a media query
  • remove the media query - problem solved
